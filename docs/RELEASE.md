# Release pipeline

Keyfount Desktop ships signed installers on tag pushes. The pipeline lives in [`.github/workflows/release.yml`](../.github/workflows/release.yml) and is driven by `tauri-apps/tauri-action`.

## What gets built

| Target              | Artefact                                            | Channel        |
| ------------------- | --------------------------------------------------- | -------------- |
| macOS Apple Silicon | `Keyfount_<version>_aarch64.dmg`, notarised         | GitHub Release |
| macOS x86_64        | `Keyfount_<version>_x86_64.dmg`, notarised          | GitHub Release |
| Windows x86_64      | `Keyfount_<version>_x64-setup.exe` + `.msi`, signed | GitHub Release |
| Auto-update         | `latest.json` (signed by the Tauri updater key)     | GitHub Release |

Mobile builds (iOS `.ipa`, Android `.aab`) are produced by `npm run build:ios` and `npm run build:android` and submitted manually to the App Store and Play Store. Those flows are intentionally **not** automated because both stores require human review and credentials we keep off CI.

## Cutting a release

```bash
# 1. Bump the version in the three locations that own it
#    (package.json, src-tauri/Cargo.toml, src-tauri/tauri.conf.json).
npm version <patch|minor|major> --no-git-tag-version

# 2. Commit the bump
git commit -am "release: vX.Y.Z"

# 3. Push the tag — this triggers the release workflow
git tag vX.Y.Z
git push origin vX.Y.Z main
```

The release workflow builds, signs, and notarises every artefact in parallel on `macos-latest` and `windows-latest`, then attaches them to a draft GitHub Release. Promote the draft once you have manually validated each installer on a clean device.

## Required GitHub secrets

| Secret                               | Purpose                                                                 |
| ------------------------------------ | ----------------------------------------------------------------------- |
| `TAURI_SIGNING_PRIVATE_KEY`          | Signs the `latest.json` so the in-app updater can verify each download. |
| `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` | Passphrase for the signing key above.                                   |
| `APPLE_CERTIFICATE`                  | Base64-encoded `.p12` Developer ID Application certificate.             |
| `APPLE_CERTIFICATE_PASSWORD`         | Passphrase for the certificate.                                         |
| `APPLE_SIGNING_IDENTITY`             | Identity string (e.g. `Developer ID Application: Keyfount (TEAMID)`).   |
| `APPLE_ID`                           | Apple ID used for notarisation.                                         |
| `APPLE_PASSWORD`                     | App-specific password for notarisation.                                 |
| `APPLE_TEAM_ID`                      | Apple Developer Team ID.                                                |

Generate the updater key with `npm run tauri signer generate` and store the public key in `src-tauri/tauri.conf.json` under `plugins.updater.pubkey`.

## Auto-update

`tauri-plugin-updater` is wired into the app and points at:

```
https://github.com/Keyfount/desktop/releases/latest/download/latest.json
```

The manifest is generated by the release workflow and signed. On startup, the app fetches the manifest, compares the version, and (silently) downloads + applies the update on the next launch when one is available.

## Rolling back

To pull a release that turned out to be broken:

1. **Mark the GitHub Release as a draft** so the auto-update endpoint stops serving it.
2. Ship a new patch release with the rollback fix.
3. The app's updater will pick the next version up automatically.

Never delete a release that has already shipped — the artefact has already been downloaded by some users, and the public update manifest needs the signature record to stay verifiable.
